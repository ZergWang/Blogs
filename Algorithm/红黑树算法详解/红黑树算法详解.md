# 为何set和map的底层使用红黑树而不是AVL树

AVL树平衡度高，代价是其调整频率也高（左右子树高度差超过1就得平衡），因此只适合查询多，增删少的场景；

红黑树平衡度低，调整频率也低（根到叶子的最长路径超过最短路径的2倍才调整），因此适合查询少，增删多的场景；
